# ADR-0031: OCR for image-only PDFs is derived text, recovered by the backfill

Date: 2026-07-12. Status: accepted.

## Context

A TMF's paper-born share — approval letters, certificates, signature pages —
arrives as scanned image-only PDFs. ADR-0022 extracts each version's text
layer at upload, so these extract to empty text and match by metadata alone;
both ADR-0022 and ADR-0030 carried OCR as the named remainder.

Two obvious shapes are wrong here. Running OCR inline at upload breaks
ADR-0022's rule that extraction never blocks an upload — OCR costs seconds
per page, not milliseconds. And an OCR sidecar service or system binary
(tesseract the executable, a cloud OCR API) is the same heavyweight runtime
dependency ADR-0030 refused for office renditions: something extra to
install, validate, and keep alive in every deployment and test environment —
and a cloud API would ship regulated documents to a third party besides.

One property matters more than either: OCR output is a *reading* of pixels,
not a fact about bytes. It belongs even further outside the audited record
than pdf.js extraction, and a search hit built on it must stay traceable to
the engine that guessed it.

## Decision

1. **OCR text lands in `document_content_text` like any extracted text** —
   content-addressed by the blob's sha256, derived, rebuildable, outside the
   audited record. Search reads it with no schema, view, or API change; the
   `extractor` column says `tesseract.js`, so an OCR-grade row is always
   distinguishable from a text-layer row in SQL.
2. **The engine is tesseract.js (WASM) with pages rendered by pdf.js through
   `@napi-rs/canvas`** — pure npm dependencies, no system binary, no service,
   nothing leaves the machine. The English traineddata (~5 MB) downloads once
   from the tesseract.js CDN into `node_modules/.cache` and is read locally
   thereafter; an air-gapped deployment pre-places that file. Rendering is
   ~144 dpi, capped at 20 pages per document.
3. **OCR runs in the backfill, never the upload request.** Upload keeps its
   fast text-layer path; a scanned PDF lands searchable by metadata, exactly
   as before. `pnpm db:extract-text` then re-reads every `application/pdf`
   whose text layer came back under 20 characters — the honest signature of
   an image-only file — and updates its row in place (`--no-ocr` skips the
   pass). A genuinely blank scan still gets its extractor updated, so nothing
   retries forever; a render or OCR *failure* leaves the text-layer row
   standing (it is still the true extraction of the bytes) and retries next
   run.
4. **The seed ships a scanned document and runs the OCR pass.** Site 002's
   IRB continuing-review approval is now an image-only PDF — a paper letter
   the coordinator scanned, generated by rasterizing its lines to a JPEG
   with no text layer at all — so a seeded stack demos search finding words
   that exist only as pixels ("twelve months"). If the traineddata download
   fails, the seed warns and completes; the backfill retries.

## Consequences

- "Search the letter, not just its filing" now covers scanned paper. Between
  upload and the next backfill run a scanned document matches by metadata
  only — at probe scale the gap is one command; a pilot would schedule it.
- The API does not label a content match as OCR-grade; the snippet may carry
  misrecognized characters. The provenance is one SQL join away
  (`extractor = 'tesseract.js'`), and surfacing it in the UI is deliberately
  deferred until a reviewer is actually misled.
- English only, 20-page cap, accuracy bounded by scan quality — stated, not
  hidden. A PDF whose genuine text layer is under 20 characters is treated
  as image-only and re-read; its OCR text replaces a near-empty truth with a
  longer guess, which is the trade the threshold makes.
- The OCR contract test is the suite's slowest and needs the traineddata
  (first run per machine downloads it); everything else in the pass is
  offline and deterministic for identical bytes and engine version.
- tesseract.js's postinstall (an opencollective banner) is deliberately not
  run: `allowBuilds` in `pnpm-workspace.yaml` pins it false.
